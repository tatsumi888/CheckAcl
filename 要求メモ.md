# アクセス権限棚卸 要求メモ

作成日: 2026-07-16

## 目的

Windowsファイルサーバ上の全ファイル・フォルダについてアクセス権限(ACL)を棚卸し、
**アクセスすべきでない人がアクセス権を持っていないこと**を確認する。

## 背景・課題

- ファイルサーバ上のファイル数が膨大で、手動での全件確認は非現実的。
- 確認作業の効率化・自動化が必須。

## 前提・規模

- 対象のフォルダ・ファイル数は **10万程度** を想定する。

## 要求事項

### R1. 権限情報の収集
- ファイルサーバ上の全ファイル・フォルダを対象に権限情報を確認できること。

### R2. 中断・再開(レジューム)
- 一定時間内に作業が完了しない場合に備え、**途中から作業を継続できる**こと。
- 進捗(処理済み箇所)を記録し、再実行時は続きから処理する。

### R3. 実行環境の制約
- **Windows標準コマンドのみ**で実現すること(追加ソフトのインストール不可)。
- セキュリティポリシーにより **.ps1 ファイルの直接実行は不可**。
  - バッチファイル経由でのPowerShell呼び出しは可能だが、**できればバッチファイルだけで完結**させたい。
  - → 採用方針: `icacls` / `dir` / `findstr` 等のcmd標準コマンドのみで構成する。

### R4. 人手確認の最小化(継承の利用)
- **継承している権限**は、その**継承の大本(おおもと)となるフォルダ**のACLを1回確認するだけで済ませる。
  - 継承の大本は必ずしもトップフォルダとは限らない。途中のフォルダで継承が切断され、
    そこに明示的な権限が設定されている場合、その配下にとっての継承元はそのフォルダとなる。
  - すなわち確認対象は「トップフォルダ+継承を切断して独自の権限体系を開始しているフォルダ」の集合であり、
    それ以外の継承のみのフォルダ・ファイルは人手確認を省略できる。
- **継承していない権限(明示的に指定された権限)は漏れなく検出**し、確認対象として抽出する。
  - icacls出力の `(I)` マーク(継承ACE)の有無で判別する。
  - 継承切断フォルダ(=下位に対する新たな継承元)は明示ACEを持つため、この抽出に必ず含まれる。

### R5. 明示的権限の内容確認(ユーザ/グループ単位)
- フォルダ・ファイルに明示的(Explicit)に指定されている権限については、
  **ユーザ/グループごとに、許可(Allow)されるアクセスと拒否(Deny)されるアクセスをすべて確認できる**こと。
  - 対象アカウント(誰に)、種別(許可/拒否)、アクセス内容(読取・書込・削除等)の3点が出力に含まれること。

### R6. アクセス不可フォルダの検出と委譲
- 実行アカウントでアクセスできないフォルダは、**存在を知り得る限り検出**して一覧化する。
  - (親フォルダが読める場合、フォルダ名までは列挙可能)
- 一覧はアクセス権を持つ担当者に**確認作業を委譲**するために使用する。

### R7. 出力形式(Excel等での確認容易性)
- **人手で確認すべき項目の出力は、Excel等でフィルタリングや一括操作が可能な形式**とする。
- CSVで出力し、Excelに読み込む運用も許容する。
  - 明示的権限は「パス / アカウント / 許可・拒否 / 権限内容」の列構成のCSVとし、
    ユーザ/グループ単位・許可/拒否単位でのフィルタリングを可能にする。

## 成果物(出力)

| ファイル | 内容 |
|---|---|
| `root_acl.txt` | トップフォルダの完全なACL(継承分の代表) |
| `explicit_acl.csv` | 明示的ACEの一覧(Path / Account / AccessType / Rights)。Excelでフィルタ可能 |
| `explicit_acl.txt` | 明示的ACEを含むicacls生出力(照合・監査証跡用) |
| `denied.csv` | アクセスできなかったフォルダ一覧(委譲用、Excel読込用) |
| `denied.txt` | 同上のicacls生エラー出力 |
| `enum_errors.txt` | フォルダ列挙時のエラー |
| `dirlist.txt` / `checkpoint.txt` | 再開用の進捗管理ファイル |

## 参考: icacls出力の読み方(R5の確認方法)

出力の各行は `<パス> <ユーザ/グループ>:(フラグ)(権限)` 形式。

- 種別: `(DENY)` が付く行は拒否、付かない行は許可(Allow)
- 継承: `(I)` が付くACEは継承、付かないものが明示的(Explicit)指定
- 主な権限表記:
  - `F` = フルコントロール / `M` = 変更 / `RX` = 読取と実行 / `R` = 読取 / `W` = 書込
  - 細分権限例: `RD`(ディレクトリ一覧/データ読取), `WD`(書込), `AD`(追加), `DE`(削除), `DC`(子削除),
    `WDAC`(アクセス許可の変更), `WO`(所有権の取得)
  - 継承方向: `(OI)`=オブジェクト継承, `(CI)`=コンテナ継承, `(IO)`=継承のみ

## 制限事項・留意点

- 中断がフォルダ処理の「途中」で発生した場合(Ctrl+C等)、チェックポイント未記録のそのフォルダは
  再実行時に最初から処理されるため、CSVに同一行が重複する可能性がある。
  → Excelの「重複の削除」機能で除去すればよい(棚卸結果への影響なし)。
- ACEの並び順(正規順序か否か)に依存しない方式(アイテム単位でicacls出力を全行解析)のため、
  明示ACEはCSVに正確なフルパス付きで漏れなく記録される。
- ジャンクション/シンボリックリンクによるループがある場合、`dir /s` の列挙に注意が必要。
- パスが260文字を超える箇所は、超過が始まるフォルダが denied 一覧に記録される(検証済み)。
  それより深い階層は列挙されないため、denied 一覧のフォルダを起点に個別確認が必要。
- 実行アカウントの読み取り権限の範囲でのみ収集可能。denied一覧で補完する。
