# CheckAcl — ファイルサーバ アクセス権限棚卸ツール

Windowsファイルサーバのアクセス権限(ACL)を棚卸するためのバッチスクリプトです。
**Windows標準コマンド(icacls / dir / findstr)のみ**で動作し、PowerShellや追加ソフトは不要です。

要求仕様の詳細は [要求メモ.md](要求メモ.md) を参照してください。

## 特徴

- **フォルダとファイルの両方が対象** — 全フォルダを1件ずつ確認するのに加え、
  各フォルダ直下の全ファイル(隠しファイル含む)もワイルドカード走査(`icacls "dir\*"`)で
  スキャンします。明示ACEを持つファイルは個別にフルパス付きでCSVに記録されます。
- **継承の利用による確認作業の最小化** — 継承されている権限は継承元フォルダの確認で代表し、
  明示的(非継承)に設定された権限だけを漏れなく抽出します。
- **中断・再開(レジューム)対応** — 処理済みフォルダ数をチェックポイントに記録。
  同じコマンドを再実行すれば続きから再開します。
- **Excelでの確認を前提としたCSV出力** — ユーザ/グループ・許可/拒否でフィルタ可能。
- **アクセス不可フォルダの検出** — 実行アカウントで読めないフォルダを一覧化し、
  権限を持つ担当者への確認委譲に使えます。

## 使い方

```bat
CheckAcl.bat <対象ルートパス> [出力フォルダ] [1回の実行で処理する最大フォルダ数]
```

例:

```bat
rem 全件処理(出力は .\acl-output)
CheckAcl.bat \\fs01\share

rem 出力先を指定し、5000フォルダ処理したら中断(再実行で再開)
CheckAcl.bat \\fs01\share D:\acl-output 5000
```

- 中断後の再開: **同じコマンドをもう一度実行**するだけです。
- 最初からやり直す場合: 出力フォルダの `dirlist.txt` と `checkpoint.txt` を削除してください。

## 出力ファイル

| ファイル | 内容 |
|---|---|
| `root_acl.txt` | トップフォルダの完全なACL(継承権限の代表) |
| `explicit_acl.csv` | 明示的ACEの一覧(Path / Account / AccessType / Rights)← **主な確認対象** |
| `explicit_acl.txt` | 明示的ACEを含むicacls生出力(監査証跡用) |
| `denied.csv` / `denied.txt` | アクセスできなかったフォルダ一覧(担当者への委譲用) |
| `enum_errors.txt` | フォルダ列挙時のエラー |
| `dirlist.txt` / `checkpoint.txt` | 再開用の進捗管理ファイル |

## 棚卸の進め方(推奨)

1. `root_acl.txt` でトップフォルダの権限を確認(継承のみのフォルダ・ファイルはこれで代表)。
2. `explicit_acl.csv` をExcelで開き、次の観点でフィルタして確認:
   - `Account` 列: `Everyone`、`Authenticated Users`、個人アカウント直指定など
   - `AccessType` 列: `Deny`(拒否設定は運用ミスの温床)
   - 継承を切断して独自権限を開始しているフォルダ(下位への新たな継承元)もここに含まれます。
3. `denied.csv` のフォルダは、アクセス権を持つ担当者に確認を依頼。

## icacls権限表記の読み方

出力の `Rights` 列は icacls の表記です。

- `(DENY)` 付き = 拒否、無し = 許可
- 主な権限: `F`=フルコントロール / `M`=変更 / `RX`=読取と実行 / `R`=読取 / `W`=書込
- 細分権限例: `RD`(一覧/読取), `WD`(書込), `AD`(追加), `DE`(削除), `DC`(子削除),
  `WDAC`(アクセス許可の変更), `WO`(所有権の取得)
- 継承方向: `(OI)`=ファイルへ継承, `(CI)`=フォルダへ継承, `(IO)`=継承のみ(自身には非適用)

## 処理時間の目安

疑似環境(フォルダ5,251+ファイル95,000 ≒ 10万アイテム、明示ACE 22件、ローカルディスク)での実測:

- 全件処理: **約6.3分**(2,000フォルダで中断→再実行で再開のシナリオ含む)
- 明示ACEの検出件数はCSVと完全一致(22/22)

ネットワーク共有(UNCパス)に対して実行する場合は、往復遅延の分これより時間がかかります。
`MaxDirsPerRun` で分割実行してください。

## 制限事項・留意点

- 実行アカウントが読み取れる範囲でのみ収集します。読めないフォルダは `denied.csv` で補完してください。
- フォルダ処理の途中でCtrl+C等により中断した場合、そのフォルダは再実行時に最初から処理されるため
  CSVに重複行が発生することがあります(完了時に自動で重複除去されます)。
- ジャンクション/シンボリックリンクによるループがあるツリーでは `dir /s` の列挙に注意してください。
- パスが260文字を超える箇所は、超過が始まるフォルダが `denied` 一覧に記録されます。
  それより深い階層は列挙されないため、該当フォルダを起点に個別確認してください。
- フォルダの所有者が実行アカウント自身の場合、拒否ACEがあってもACLは読めるため denied には載りません。
